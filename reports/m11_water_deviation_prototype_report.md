# M11 Water Deviation Prototype Report

## 实现目标

本轮在 M11 prototype 分支中为状态总览的水质区域增加“目标值 / Δ偏离值 / 风险标签”显示，帮助玩家理解水质参数应围绕目标区间稳定，而不是越高越好。

本轮只做显示型 prototype，没有修改水质模拟公式、保存系统、Godot scene 或正式 UI 美术。

## 修改文件列表

- `scenes/ui/StatusPanel.gd`
- `reports/m11_water_deviation_prototype_report.md`

## 目标值表

这些数值是 prototype tuning values，只用于游戏教学和 UI 验证，不是专业饲养建议。

| 参数 | 目标值 | 显示用途 |
| --- | ---: | --- |
| 温度 | 25.0 | 对比当前温度并显示 Δ |
| 盐度 | 35.0 | 对比当前盐度并显示 Δ |
| pH | 8.20 | 对比当前 pH 并显示 Δ |
| NO3 | 2.00 | 解释过低、正常、偏高 |
| PO4 | 0.030 | 解释过低、正常、偏高 |
| KH | 8.3 | 对比当前 KH 并显示 Δ |
| Ca | 430 | 对比当前 Ca 并显示 Δ |

## 阈值规则

- 温度：abs(Δ) <= 0.5 正常；<= 1.5 偏高/偏低；否则 危险偏高/危险偏低
- 盐度：abs(Δ) <= 1.0 正常；<= 3.0 偏高/偏低；否则 危险偏高/危险偏低
- pH：abs(Δ) <= 0.15 正常；<= 0.35 偏高/偏低；否则 危险偏高/危险偏低
- NO3：低于 0.5 过低/过净；1.0-10.0 正常；10.0-20.0 注意偏高；高于 20.0 偏高
- PO4：低于 0.005 过低/过净；0.010-0.100 正常；0.100-0.200 注意偏高；高于 0.200 偏高
- KH：abs(Δ) <= 0.5 正常；<= 1.0 偏高/偏低；否则 危险偏高/危险偏低
- Ca：abs(Δ) <= 30 正常；<= 60 偏高/偏低；否则 危险偏高/危险偏低

## 玩家教学意义

偏离值让玩家直接看到“当前值距离目标值有多远”。这比单独显示当前数值更容易理解水质管理：温度、盐度、pH、KH、Ca 需要稳定在目标附近；NO3 / PO4 太低也可能代表系统过净，不一定是好事。

## 为什么不是越高越好

水质参数不是简单的数值成长。过高可能带来压力或污染风险，过低也可能意味着营养不足、缓冲不足或系统不稳定。prototype UI 用“Δ”和“过低/偏高/正常”把这种双向风险显性化。

## 出门前缓冲阈值的后续扩展

未来可以在正式设计中加入“出门前缓冲”概念，例如玩家预计多天离线时，提前把某些会自然恶化的参数调整到更稳的位置。但 KH、盐度、pH 等变化不应鼓励剧烈调整，后续需要结合变化速度、生物耐受和操作风险一起设计。

## 风险点

- 目标值和阈值目前是 UI prototype tuning values，后续正式版需要和水质系统目标、数值平衡、玩家提示文本统一。

- “过低/过净”等标签可能需要后续文案打磨，避免玩家误解成专业饲养建议。
