# ReefIdle 视觉 QA 协议 v1.0

## 目标

将人工视觉审查从"逐项零散确认"升级为"批量聚合审查"。执行层每轮至少收集 10 个可见问题后一次性提交。

## 核心规则

1. **每轮最少 10 个问题** — 不足 10 个时继续收集，不提交
2. **批量提交** — 10+ 问题一次性列出，附带截图和位置
3. **分类聚合** — 同类问题合并为一个条目
4. **不复述代码** — 只描述玩家可见的现象
5. **逐个截图** — 每个问题至少一张截图

## 视觉 QA 检查清单

### 1. UI 布局与裁剪

- [ ] 左侧面板是否裁剪或越界
- [ ] 右侧面板是否裁剪或越界
- [ ] 底部面板是否完整可见
- [ ] 弹窗/对话框是否在屏幕范围内
- [ ] ScrollContainer 是否正确裁剪内容
- [ ] 不同窗口尺寸下布局是否稳定（1280x720 / 1920x1080）

### 2. 文字与标签

- [ ] 所有标签文字完整显示，无截断
- [ ] 中文/英文排版正常，无乱码
- [ ] 无 Unicode 转义残留（\uXXXX）
- [ ] 按钮文字清晰可读
- [ ] 数字格式化正确（RP 数值、百分比）
- [ ] 空状态时有合理的占位文字，非空白

### 3. 颜色与对比度

- [ ] 文字与背景对比度足够
- [ ] 状态颜色语义正确（绿=好，红=警告，黄=注意）
- [ ] 暗色主题下所有元素可见
- [ ] 按钮不同状态（normal/hover/pressed/disabled）视觉可区分

### 4. 交互反馈

- [ ] 按钮点击有视觉反馈
- [ ] 购买操作后有确认提示或动画
- [ ] 设备开关状态有颜色/图标区分
- [ ] 维护操作后指标变化可见
- [ ] 错误操作有提示（非静默失败）

### 5. 数据一致性

- [ ] 购买后库存数量更新
- [ ] 设备开关后状态面板同步
- [ ] 维护后水质指标实时刷新
- [ ] RP 变化与操作关联
- [ ] 时间线条目与实际操作一一对应

### 6. 空状态与边界

- [ ] 无生物时 UI 正常
- [ ] 无设备时 UI 正常
- [ ] RP=0 时 UI 正常
- [ ] 所有设备关闭时状态正常
- [ ] Reset 后所有 UI 恢复到默认状态

### 7. 时间线与事件日志

- [ ] 时间线条目格式统一
- [ ] 新事件出现在列表顶部或底部（保持一致）
- [ ] 时间线不溢出、不空白
- [ ] 事件文字描述完整可读

## 视觉 QA 提交格式

每轮聚合后，以以下格式一次性提交：

```markdown
## Visual QA Round [N]
Date: YYYY-MM-DD HH:MM
Screenshots: reports/visual_qa/round_N/

### Issues Found: [M]

#### Issue 1: [简短标题]
- **位置**: [UI 面板/按钮/标签名称]
- **截图**: round_N/issue_01.png
- **现象**: [玩家看到的实际效果]
- **期望**: [应该看到的效果]
- **严重度**: [BLOCKER / HIGH / MEDIUM / LOW]
- **建议**: [修复方向，可选]

#### Issue 2: ...
...
```

## 严重度定义

| 级别 | 定义 | 示例 |
|------|------|------|
| BLOCKER | 玩家无法继续操作 | 购买按钮不可见、面板全白 |
| HIGH | 核心体验受损 | RP 显示错误、设备开关无效 |
| MEDIUM | 有影响但可绕过 | 文案截断、颜色不一致 |
| LOW | 视觉瑕疵 | 间距不完美、圆角不一致 |

## 禁止行为

- 禁止每发现 1 个问题就提交一次
- 禁止无截图的口头描述
- 禁止把 UI 显示问题归因于"用户屏幕问题"
- 禁止跳过边界状态检查
- 禁止用"看起来 ok"代替逐项检查
