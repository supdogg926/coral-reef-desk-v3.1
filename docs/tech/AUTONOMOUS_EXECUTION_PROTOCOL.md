# ReefIdle 自治执行协议 (AEP v1.0)

## 目标

把 ReefIdle 执行层（Codex / Claude Code / 后续 AI 执行工具）从"每一步都要确认"升级为"护栏内自治执行"。

## 适用范围

本协议适用所有在 `CoralReefIdleV3` 仓库工作的 AI 执行工具。

## 自治级别定义

### Level 0 — 完全确认模式（已废弃）

每个操作都要人工确认。仅用于以下紧急情况：
- 检测到 save 系统损坏
- Godot 项目文件（.godot/）被意外修改
- 仓库处于 detached HEAD 状态

### Level 1 — 标准自治模式（当前默认）

在栅栏内的操作可以连续执行，不需逐操作确认。

**自治窗口内的操作**：
- 文件读写（.gd / .tscn / .md / .ps1 / .py / .json）
- Godot headless 测试/验收运行
- Git 只读操作
- 工具脚本运行
- 验收报告生成

**自治窗口边界**：
- 一次最多 15 个写操作
- 一次最多 10 个 Godot 运行
- 每轮视觉 QA 至少聚合 10 个问题
- 窗口关闭时汇报进度

### Level 2 — 全自治模式（M12+ 目标）

Level 1 规则 + commit/push 也可以在满足双 PASS 条件下自动执行。需要 M12 完成后再激活。

## 决策树

```
收到任务
  │
  ├─ 是否在禁止操作列表？ → STOP，汇报原因
  │
  ├─ 是否在 STOP CONDITIONS 列表？ → 停止等待确认
  │
  ├─ 是否在 NO-ASK LIST？ → 直接执行
  │
  ├─ 当前窗口写操作 ≥ 15？ → 汇报进度，等待确认
  │
  ├─ 当前窗口 Godot 运行 ≥ 10？ → 汇报状态，等待确认
  │
  └─ 否 → 继续自治执行
```

## STOP CONDITIONS（必须停止）

以下操作执行层必须停止并等待用户确认：

| 操作 | 原因 |
|------|------|
| `git commit` | 用户对 commit message 和内容有最终审查权 |
| `git push` | 推送影响远端，不可自动 |
| `git tag -d` / 创建新 tag | Tag 是里程碑，不可自动 |
| 删除 `.gd` / `.tscn` 文件 | 源码删除不可逆 |
| 修改 `SaveSystem` | 存档系统敏感，需人工审查 |
| 修改 `project.godot` | 项目配置敏感 |
| 跨分支操作 | 分支切换影响全局 |
| 重写 Git 历史 | `rebase -i` / `reset --hard` / `push --force` |

## NO-ASK LIST（禁止询问的冗余确认）

执行层不得因以下原因停下来询问：

- "我可以继续下一步吗？" → 自治窗口内直接执行
- "我可以运行测试吗？" → 直接运行，失败再汇报
- "我可以读取这个文件吗？" → 直接读取
- "看起来没问题，要继续吗？" → 直接继续
- "需要我运行验收脚本吗？" → 如果还没跑过，直接跑
- 单次工具脚本结果 → 除非失败，聚合汇报
- "我可以编辑这个文件吗？" → 在 Level 1 窗口内直接编辑

## 批量处理规则

### 写操作批处理

- 同一自治窗口内最多 15 个写操作
- 超过 15 个时，汇报当前进度列表，等待确认
- 写操作包括：文件创建、编辑、删除

### Godot 运行批处理

- 同一自治窗口内最多 10 个 Godot headless 运行
- 超过 10 个时，汇报通过的测试列表，等待确认

### 视觉 QA 批处理

- 每轮至少聚合 **10 个可见问题**才能提交给用户
- 禁止逐个截图、逐个确认
- 格式：问题编号 + 截图位置 + 现象描述 + 建议修复方向

## 失败升级规则

| 失败类型 | 处理 |
|----------|------|
| 单个 GDScript 测试 FAIL | 停止当前任务，汇报失败断言详情 |
| 验收脚本 FAIL | 立即停止，不回退代码，不自动修复 |
| Godot 无法启动 | 检查路径，重试一次；仍失败则停止 |
| Git 冲突 | 停止，汇报冲突文件列表 |
| 文件写入权限错误 | 停止，汇报路径 |
